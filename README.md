# kotlin-racingcar-precourse

## 구현할 기능 목록 정리

### 입력 기능
- [ ] n대의 자동차 이름을 쉼표(,)기준으로 구분하여 입력
- [ ] 시도할 횟수를 숫자로 입력

### 랜덤 넘버 기능
- [ ] 랜덤 API를 통해 0~9까지의 랜덤 넘버 생성

### 자동차 이름 구분
- [ ] 입력받은 자동차 이름을 리스트로 변환

### 레이싱 기능
- [ ] 랜덤 번호가 4 이상인지 Boolean으로 확인
- [ ] true일 경우 카운트 증가
- [ ] 입력된 시도 횟수만큼 반복
- [ ] 최댓값으로 우승자를 확인 (2명 이상일 경우 쉼표(,)로 구분)

### 출력 기능
- [ ] 입력 문구 출력
- [ ] 시도 횟수 문구 출력
- [ ] 차수별 자동차 이름과 현재 위치를 대시(-) 로 실행 결과 출력
- [ ] 최종 우승자 문구 출력

### 예외 처리 기능
- [ ] 구분자가 쉼표(,)가 아니면 예외 처리
- [ ] 구분자 기준으로 자동차 이름이 6자 이상이면 예외 처리
- [ ] 자동차 이름에 공백이 포함되어 있으면 예외 처리
- [ ] 횟수 입력이 Int타입을 초과하면 예외 처리
- [ ] 횟수 입력이 양수가 아닐 경우 예외 처리
- [ ] 횟수 입력이 0일 경우 예외 처리
- [ ] 입력이 공백일 경우 예외 처리

### 1주차 코드 리뷰
- [ ] 의존성을 낮추기 위해 생성자를 주입
- [ ] 클래스 내에서는 관련 로직만 사용
- [ ] 함수 길이 최소화
- [ ] 들여쓰기 2까지만 활용
- [ ] 필요한 공백만 활용

### 1주차 공통 피드백
- [ ] 요구 사항을 정확하게 준수한다
- [ ] 기본적인 Git 명령어를 숙지한다
- [ ] Git으로 관리할 자원을 고려한다
- [ ] 커밋 메시지를 의미 있게 작성한다
- [ ] 커밋 메시지에 이슈 또는 풀 리퀘스트 번호를 포함하지 않는다
- [ ] 풀 리퀘스트를 만든 후에는 닫지 말고 추가 커밋을 한다
- [ ] 오류를 찾을 때 출력 함수 대신 디버거를 사용한다
- [ ] 이름을 통해 의도를 드러낸다
- [ ] 축약하지 않는다
- [ ] 공백도 코딩 컨벤션이다
- [ ] 공백 라인을 의미 있게 사용한다
- [ ] 스페이스와 탭을 혼용하지 않는다
- [ ] 코드 포매팅을 사용한다
- [ ] Kotlin에서 제공하는 API를 적극 활용한다
- [ ] 배열 대신 컬렉션을 사용한다
