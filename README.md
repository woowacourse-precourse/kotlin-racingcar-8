# kotlin-racingcar-precourse

# [02주차] 자동차 경주

## 기능 구현 목록

### 자동차 이름들을 입력 받아서 자동차 목록을 만든다
- 쉼표(,) 기준으로 분리해서 `Car` 객체로 변환한다
- 자동차 이름은 5자를 초과할 수 없다
- 잘못된 이름(빈 문자열, 5자 초과 등)이 들어오면 `IllegalArgumentException`을 던지고 프로그램을 종료함
- 이름 중복을 허용해야 하는지 애매하지만 일단은 허용한다고 가정 =>
    - 중복을 막으려면 `Set`을 써야 하지만 요구사항에 명시가 없고,  
      실제 게임 로직에 영향이 없다고 판단해서 우선 허용함
    - 추후 확장 시 중복 검증 로직을 `CarFactory`에 추가할 수도 있음.

### 시도할 총 이동 횟수를 입력받는다
- 문자열 입력을 `Int`로 변환한다
- 음수거나 숫자가 아니면 `IllegalArgumentException`을 던진다
- 0회 경주도 가능한지 애매한데, 현재는 0이면 그냥 이동 없이 우승자만 계산되도록 둘 예정
- 0회 입력 시 에러로 볼 수도 있지만,  
  “0번 시도된 상태”도 논리적으로 가능하다고 생각
  실제로는 아무 이동이 없으므로 모든 자동차가 동점 → 전부 우승자로 처리.

### 한 번의 시도(라운드)를 진행한다
- 각 자동차마다 0~9 사이의 랜덤 숫자를 구한다 (`Randoms.pickNumberInRange(0, 9)`)
- 랜덤 숫자가 4 이상이면 자동차는 한 칸 전진한다
- 전진은 내부적으로 `position` 값을 +1 하는 방식으로 관리한다
- 랜덤을 바로 호출하면 테스트하기 어려움 →  
  `tryMove(randomNumber)` 형태로 분리해서 랜덤 없이도 테스트 가능하게 설계함

### 현재 라운드의 결과를 출력 형식에 맞게 보여준다
- 예: `pobi : ---`
- `position` 수만큼 `"-"` 출력
- 출력 형식은 콜론(:) 뒤에 공백 하나 포함
- 포맷 상수화 고민했지만 단순해서 문자열로 처리
  라운드 사이 빈 줄은 예시 출력 따라 넣기로 결정

### 전체 게임을 진행한다
- 사용자가 입력한 횟수만큼 라운드를 반복 실행
- 각 라운드마다 모든 자동차 이동 후 상태 출력
- `for` 대신 `repeat` 사용 → 코틀린다운 코드
  `Application`이 입출력만 담당하고, 핵심 로직은 `RacingGame`에서 수행하도록 분리하였음

### 최종 우승자를 계산한다
- 가장 멀리 간 position을 찾고
- 그 값과 같은 자동차 이름들을 우승자로 판정
- 우승자가 여러 명이면 쉼표(", ")로 구분
- 예: `최종 우승자 : pobi, jun`
- 전부 같은 위치일 경우 전부 우승자로 처리.  
  빈 리스트일 가능성은 거의 없지만 방어 코드로 기본값 0 설정

### 예외 처리
- 잘못된 입력 시 `IllegalArgumentException` 던지고 종료
- 예외 메시지는 단순하게 유지 (테스트 결과 간결성 유지 목적)
- catch해서 재입력받을 수도 있지만  
  요구사항에서 “예외 발생 시 종료” 명시되어 있으므로 그대로 종료

### 테스트 코드 작성
- 이름 5자 초과 시 예외 발생 테스트
- 랜덤값 4 이상일 때 이동, 미만이면 정지 테스트
- 우승자 다수일 경우 쉼표로 구분되는지 테스트
- I/O(`Console.readLine`)은 제외, 로직 중심 테스트만 진행
- 랜덤 테스트 불가능 → `tryMove()` 단위로 테스트
  AssertJ는 나중에 도입하고 지금은 기본 JUnit으로 연습
